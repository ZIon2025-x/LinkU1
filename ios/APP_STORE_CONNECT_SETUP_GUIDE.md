# App Store Connect 配置完整指南

## 📋 概述

App Store Connect 是 Apple 提供的平台，用于管理应用提交、审核和发布。本指南将详细说明如何配置 App Store Connect。

---

## 🔑 第一步：登录 App Store Connect

### 1. 访问 App Store Connect

1. **打开浏览器**，访问：https://appstoreconnect.apple.com
2. **使用 Apple Developer 账号登录**
   - 必须使用已注册 **Apple Developer Program** 的账号（$99/年）
   - 如果还没有，需要先注册：https://developer.apple.com/programs/

### 2. 确认账号状态

登录后，确认：
- ✅ 账号状态为 **Active**（活跃）
- ✅ 已支付年度费用
- ✅ 有权限创建应用

---

## 📱 第二步：创建 App 记录

### 1. 进入"我的 App"

1. 登录后，点击左侧菜单 **"我的 App"**（My Apps）
2. 点击右上角 **"+"** 按钮
3. 选择 **"新 App"**（New App）

### 2. 填写基本信息

**必填信息**：

| 字段 | 填写内容 | 说明 |
|------|---------|------|
| **平台** | iOS | 选择 iOS |
| **名称** | Link²Ur | 应用显示名称（最多 30 个字符） |
| **主要语言** | 英语（English） | 应用的主要语言 |
| **Bundle ID** | com.link2ur | 选择已注册的 Bundle Identifier |
| **SKU** | link2ur-ios-001 | 唯一标识符（不能更改，建议格式：应用名-平台-编号） |

**SKU 说明**：
- SKU（Stock Keeping Unit）是内部标识符
- 不会显示给用户
- 一旦创建不能更改
- 建议格式：`link2ur-ios-001`

### 3. 创建 App 记录

1. 填写完所有信息后
2. 点击 **"创建"**（Create）
3. 等待创建完成（通常几秒钟）

---

## 📝 第三步：配置应用信息

### 1. 进入应用详情页

创建成功后，会进入应用详情页，包含以下标签：
- **App 信息**（App Information）
- **定价和可用性**（Pricing and Availability）
- **准备提交**（Prepare for Submission）
- **TestFlight**（测试）
- **App 分析**（Analytics）

### 2. 配置 App 信息

点击 **"App 信息"**（App Information）标签：

#### 基本信息

| 字段 | 填写内容 | 说明 |
|------|---------|------|
| **名称** | Link²Ur | 应用名称（最多 30 个字符） |
| **副标题** | Link to You | 可选，显示在名称下方（最多 30 个字符） |
| **类别** | 主要：商务（Business）<br>次要：社交网络（Social Networking） | 选择应用类别 |
| **内容版权** | © 2025 Link²Ur | 版权信息 |
| **年龄分级** | 需要完成内容问卷 | 点击"编辑"完成问卷 |

#### 年龄分级问卷

1. 点击 **"编辑"** 进入年龄分级问卷
2. 根据应用内容如实回答：
   - **暴力内容**：无或轻微
   - **性内容或裸体**：无
   - **赌博和竞赛**：无
   - **酒精、烟草或毒品**：无
   - **医疗/治疗信息**：无
   - **模拟赌博**：无
   - **恐怖/惊悚题材**：无
   - **频繁/强烈的成人/性暗示题材**：无
   - **频繁/强烈的现实暴力**：无
   - **频繁/强烈的卡通或幻想暴力**：无
   - **频繁/强烈的亵渎或粗俗幽默**：无
   - **无限制的网页访问**：无
   - **赌博和竞赛**：无
3. 完成后点击 **"保存"**

#### 应用本地化（可选）

如果需要支持多语言：
1. 点击 **"+"** 添加语言
2. 选择语言（如：简体中文）
3. 填写对应语言的应用信息

## 💰 第四步：配置定价和可用性

### 1. 进入定价和可用性

点击 **"定价和可用性"**（Pricing and Availability）标签

### 2. 设置价格

| 选项 | 设置 | 说明 |
|------|------|------|
| **价格** | 免费（Free） | 选择免费 |
| **可用地区** | 所有国家或地区 | 或选择特定国家/地区 |

### 3. 设置可用性

- **可用性**：选择 **"所有国家或地区"** 或特定国家/地区
- **自动续订订阅**：如果应用有订阅功能，需要配置

## 📸 第五步：准备应用材料

### 1. 进入准备提交

点击 **"准备提交"**（Prepare for Submission）标签

### 2. 上传应用图标

**要求**：
- 尺寸：**1024 x 1024 像素**
- 格式：PNG 或 JPEG
- 无透明度
- 无圆角（Apple 会自动添加）
- 符合设计指南

**步骤**：
1. 点击 **"选择文件"** 或拖拽图标文件
2. 上传图标
3. 等待处理完成

### 3. 上传截图

**要求**：
- **至少需要 3 张截图**
- 不同设备尺寸需要不同的截图：

| 设备 | 尺寸 | 说明 |
|------|------|------|
| iPhone 6.7" | 1284 x 2778 像素 | iPhone 14 Pro Max |
| iPhone 6.5" | 1242 x 2688 像素 | iPhone 11 Pro Max |
| iPhone 5.5" | 1242 x 2208 像素 | iPhone 8 Plus |
| iPad Pro 12.9" | 2048 x 2732 像素 | iPad Pro |

**步骤**：
1. 选择设备尺寸
2. 点击 **"+"** 添加截图
3. 上传截图（至少 3 张）
4. 可以拖拽调整顺序

**截图建议**：
- 第一张：主界面或核心功能
- 第二张：主要功能展示
- 第三张：其他重要功能
- 展示应用的核心价值和功能

### 4. 填写应用描述

#### 简短描述（Subtitle）

- **最多 170 个字符**
- 显示在应用名称下方
- 建议：简洁描述应用核心价值

**示例**：
```
Link²Ur - 连接学生与本地服务的平台。发布任务、购买二手商品、参与社区讨论。
```

#### 完整描述（Description）

- **最多 4000 个字符**
- 详细描述应用功能
- 可以使用列表、换行等格式

**示例**：
```
Link²Ur 是一个专为学生设计的本地服务平台，帮助您：

• 发布和接受任务
• 浏览和购买二手商品
• 参与社区论坛讨论
• 连接本地服务提供者

主要功能：
- 任务市场：发布任务、接受任务、管理任务
- 跳蚤市场：浏览商品、发布商品、议价交易
- 社区论坛：发帖、回帖、点赞、收藏
- 实时消息：与用户实时聊天
- 个人中心：管理任务、查看钱包、设置偏好

安全可靠：
- 学生身份验证
- 安全的支付系统
- 完善的隐私保护

立即下载，开始使用 Link²Ur！
```

#### 关键词（Keywords）⚠️ **重要**

- **最多 100 个字符**
- **用逗号分隔，不能有空格**
- **必须包含 "link2ur"** 以确保用户搜索能找到应用

**推荐关键词**：
```
link2ur,link to you,link2u,学生,任务,服务,跳蚤市场,论坛,社区,本地服务,兼职,二手,交易
```

**说明**：
- 必须包含 `link2ur`（普通数字2），因为应用名称是 "Link²Ur"（上标2）
- 用户搜索 "link2ur" 时能找到应用
- 详见：`ios/APP_STORE_SEARCH_OPTIMIZATION.md`

#### 推广文本（Promotional Text）

- **最多 170 个字符**
- **可以随时更新，无需审核**
- 用于推广新功能或活动

**示例**：
```
新功能上线！现在支持 Apple Pay 支付，更安全、更便捷。
```

### 5. 填写支持信息

| 字段 | 填写内容 | 说明 |
|------|---------|------|
| **支持 URL** | https://www.link2ur.com/support | 支持网站 |
| **营销网站 URL** | https://www.link2ur.com | 营销网站（可选） |
| **隐私政策 URL** | https://www.link2ur.com/privacy | **必需** |
| **联系信息** | | |
| - 联系邮箱 | info@link2ur.com | 支持邮箱 |
| - 联系电话 | （可选） | 支持电话 |
| - 联系地址 | A3, B16 9NB, UK | 联系地址 |

## 🔒 第六步：配置隐私信息

### 1. 进入 App 隐私

在应用详情页，点击 **"App 隐私"**（App Privacy）标签

### 2. 填写隐私实践

根据应用实际功能，如实填写：

#### 收集的数据类型

**必须选择的数据类型**：

1. **联系信息（Contact Info）**
   - ✅ 姓名（Name）
   - ✅ 电子邮件地址（Email Address）
   - ✅ 电话号码（Phone Number，可选）

2. **用户内容（User Content）**
   - ✅ 用户生成的内容（User-Generated Content）
   - ✅ 照片或视频（Photos or Videos）

3. **标识符（Identifiers）**
   - ✅ 用户ID（User ID）
   - ✅ 设备ID（Device ID）

4. **位置信息（Location）**
   - ✅ 精确位置（Precise Location，可选）
   - ✅ 粗略位置（Coarse Location，可选）

5. **使用数据（Usage Data）**
   - ✅ 产品交互（Product Interaction）

6. **诊断数据（Diagnostics）**
   - ✅ 崩溃数据（Crash Data）
   - ✅ 性能数据（Performance Data）
   - ✅ 其他诊断数据（Other Diagnostic Data）

#### 数据使用目的

**必须选择的使用目的**：

- ✅ 应用功能（App Functionality）
- ✅ 分析（Analytics）
- ✅ 应用改进（App Improvements）
- ✅ 个性化（Personalization）
- ✅ 其他目的（Other Purposes）：安全、合规

#### 数据共享

**与第三方共享的数据**：

1. **云服务提供商**
   - 第三方名称：**Railway**
   - 共享的数据：所有用户数据
   - 用途：数据存储、服务器托管

2. **支付服务提供商**
   - 第三方名称：**Stripe**
   - 共享的数据：支付相关信息
   - 用途：处理支付交易

#### 追踪（Tracking）

- **是否用于追踪**：**否**
  - 应用不使用数据来追踪用户在其他公司的应用和网站上的活动

### 3. 保存隐私信息

填写完成后，点击 **"保存"**

**详细指南**：详见 `ios/APP_STORE_SUBMISSION_CHECKLIST.md` 第 2 节"法律合规性"

## 📤 第七步：上传构建版本

### 1. 在 Xcode 中构建 Archive

1. **打开 Xcode**
2. **打开项目**：`ios/link2ur.xcodeproj`
3. **选择设备**：选择 "Any iOS Device" 或 "Generic iOS Device"
4. **构建 Archive**：
   - 菜单：**Product → Archive**
   - 或快捷键：`⌘ + B`（先构建），然后 `Product → Archive`
5. **等待构建完成**

### 2. 上传到 App Store Connect

1. **Archive 完成后**，会自动打开 Organizer 窗口
2. **选择构建版本**，点击 **"Distribute App"**
3. **选择分发方式**：
   - 选择 **"App Store Connect"**
   - 点击 **"Next"**
4. **选择分发选项**：
   - 选择 **"Upload"**（上传）
   - 点击 **"Next"**
5. **选择分发内容**：
   - 选择 **"Automatically manage signing"**（如果使用自动签名）
   - 或选择 **"Manually manage signing"**（如果使用手动签名）
   - 点击 **"Next"**
6. **确认信息**：
   - 检查应用信息
   - 点击 **"Upload"**
7. **等待上传完成**

### 3. 在 App Store Connect 中处理构建版本

1. **上传完成后**（可能需要几分钟到几小时）
2. **返回 App Store Connect**
3. **进入应用详情页** → **"准备提交"** 标签
4. **在"构建版本"部分**：
   - 点击 **"+"** 选择构建版本
   - 选择刚才上传的构建版本
   - 等待处理完成（可能需要几分钟）

### 1. 完成所有必填项

- [x] 应用图标已上传

- [x] 隐私信息已填写

### 2. 填写审核信息

在 **"准备提交"** 标签底部，填写审核信息：

#### 联系信息

- **审核备注**（可选）：说明应用功能或特殊要求
- **演示账户**（如果应用需要登录）：
  - 用户名：test@link2ur.com
  - 密码：Test123456
  - 说明：测试账户，用于审核

#### 版本发布

- **自动发布**：选择 **"手动发布"** 或 **"自动发布"**
  - 手动发布：审核通过后，需要手动点击发布
  - 自动发布：审核通过后，自动发布到 App Store

### 3. 提交审核

1. **检查所有信息**无误
2. **点击右上角 "提交以供审核"**（Submit for Review）
3. **确认提交**
4. **等待审核**（通常 24-48 小时）

## 📊 第九步：跟踪审核状态

### 审核状态

提交后，可以在 App Store Connect 中查看审核状态：

| 状态 | 说明 |
|------|------|
| **等待审核**（Waiting for Review） | 已提交，等待 Apple 审核 |
| **审核中**（In Review） | Apple 正在审核 |
| **待发布**（Pending Release） | 审核通过，等待发布 |
| **已发布**（Ready for Sale） | 已在 App Store 上架 |
| **被拒绝**（Rejected） | 审核被拒绝，需要修复问题 |

### 审核时间

- **通常**：24-48 小时
- **可能更长**：如果遇到问题或需要更多信息
- **加急审核**：可以申请，但需要合理理由

## 🔧 常见问题

### Q: Bundle ID 在哪里注册？

**A**: 
1. 登录 Apple Developer：https://developer.apple.com/account
2. 进入 Certificates, Identifiers & Profiles
3. 选择 Identifiers → 点击 "+"
4. 选择 App IDs → Continue
5. 填写信息：
   - Description: Link²Ur
   - Bundle ID: com.link2ur
6. 点击 Register

### Q: 关键词可以随时修改吗？

**A**: 
- 可以，但需要提交新版本或更新应用信息
- 可能需要重新审核（取决于修改内容）

### Q: 截图可以随时更换吗？

**A**: 
- 可以，但需要提交新版本
- 需要重新审核

### Q: 应用被拒绝怎么办？

**A**: 
1. 查看拒绝原因（在 App Store Connect 中）
2. 修复问题
3. 重新提交审核

### Q: 可以加速审核吗？

**A**: 
- 可以申请加急审核
- 需要合理理由（如：关键 bug 修复）
- 每年有次数限制
