关键设计决策

✅ group_messages 独立表

不与现有 Message 表混用，避免巨表问题。独立索引：(group_id, created_at DESC) 保证群聊查询性能。

✅ 游标分页

复用任务聊天的 cursor-based 分页模式（last_read_message_id），适合消息流场景。

✅ 已读追踪

复用 MessageReadCursor 模式：每用户每群一条记录，记录最后已读消息 ID，而非每条消息标 read。

✅ mentions 用 JSON

["user_id_1", "user_id_2", "all"] — 简单高效，支持 @全体。前端解析渲染高亮。

✅ meta 字段灵活存储

分享的任务/商品/活动卡片信息存在 meta JSON 中，不同 message_type 对应不同 schema。

✅ 软删除 + 状态管理

成员用 status（active/banned/left），群组用 is_active，不做物理删除。

索引策略

-- 核心查询索引
CREATE INDEX ix_group_msg_group_created ON group_messages(group_id, created_at DESC);
CREATE INDEX ix_group_msg_group_pinned ON group_messages(group_id) WHERE is_pinned = true;
CREATE INDEX ix_group_members_user ON group_members(user_id) WHERE status = 'active';
CREATE INDEX ix_group_members_group ON group_members(group_id, role);
CREATE UNIQUE INDEX ux_group_reads ON group_message_reads(group_id, user_id);
CREATE INDEX ix_groups_category ON chat_groups(category) WHERE is_active = true;
CREATE INDEX ix_groups_tags ON chat_groups USING gin(tags);
