· 评价人立场: 评论前一个圆形 icon —— 👍 绿色表示该用户给这个竞品 upvote, 👎 红色表示 downvote (取自现有 vote_type 字段,不变)

· 点卡片跳转目标:可以继续是榜单条目详情页(或 leaderboard 页面定位到该 item)。

实现思路

Flutter 改动:
1. _ServiceReviewCard 已用海报式 + serviceType 切配色 ✅ (已完成)
2. _CompetitorReviewCard:有 targetItem.thumbnail → 海报式(同 service review 结构),把现有 UP/DOWN row 替换为前置 stance-icon(👍/👎,来自 item.voteType);右上角加 _RankBadge(#1 金 / #2 银 / #3 铜 / 其他黑底)
3. 无 thumbnail → 保留现在的白底引用框样式作 fallback

后端改动 (_fetch_competitor_reviews):
· 需要算 rank 字段 —— 该 LeaderboardItem 在所属 CustomLeaderboard 内的排名(按 upvotes-downvotes desc 或现有排序逻辑)
· 实现选项: ① 子查询用 ROW_NUMBER() OVER (PARTITION BY leaderboard_id ORDER BY upvotes-downvotes DESC) ② 在 Python 层 group-by leaderboard 后排序赋名次
· 加到 target_item.rank,前端 TargetItemBrief.rank: int?

建议先做:UI 改造在前(用占位 rank=null 时不显示角标),后端 rank 计算在后 —— 后端 SQL window function 要小心成本,值得单独评估。

实现思路 — 帖子卡 + 达人卡

帖子卡 (_PostCard):
· 统一海报式 — 无论有无首图,都用 Stack(fit: StackFit.expand, ...) 海报布局
· item.hasImages = true → 首图 AsyncImageView 占满做背景
· item.hasImages = false → 用 category 渐变(参考 ServiceCategoryHelper.getGradient 或自建 post category gradient map) + item.categoryIcon emoji 大字(fontSize: 90, opacity: 0.5) 占位 —— 跟达人卡 fallback 风格一致
· 顶部加 LinearGradient(0.35→0→0.45→0.85) veil 保证 overlay 文字可读
· 顶部 overlay: 左上 category badge (按 category 染色), 右上 stats chip (❤ likeCount · 💬 commentCount)
· 底部 overlay: title (2 行) + desc (2 行) + 用户行 (AvatarView + userName · 时间) + linkedItem tag (有则显)
· cardRadius 沿用 _kDiscoveryCardRadius,aspect-ratio 改 3:4 (跟现在的达人卡一致,masonry 高度差异更舒服)

达人卡 (_ExpertCard):
· 现有 cover (3:4) 已经存在,只需把下半白底信息区也合进 cover overlay
· cover 缺失 → 沿用 ServiceCategoryHelper.getGradient(category) 渐变 + 类别 emoji 大字 (透明度 0.5),Flutter 用 Center(child: Text(emoji, fontSize: 90, color: white.withOpacity(0.5)))
· 头像从 cover 左下移到 content overlay 内,跟名字成 head-row;尺寸 38-40px + 白边 + 阴影
· 技能 chip 改半透白底 (Colors.white.withOpacity(0.22)) + 边框 0.22 alpha,不再用 Color(0xFFDBEAFE) 蓝底
· 营业中/休息中 pill: 营业中绿色 0.92, 休息中灰色 0.88
· 评分 0 时显示 "★ 新 · 刚开业" tag (不显示 "完单 0")
· 推荐理由 expertReasonCode 在海报式里塞不下,**移到 cover 顶部右侧或省略**(masonry 卡太挤);可选保留在长按 / 详情页

无需后端字段改动 — 字段全部沿用现有 DiscoveryFeedItem。
渐进上线: 先做帖子卡 (改动小,fallback 安全),再做达人卡 (信息密度变化大,需要看 masonry 流整体节奏)。
