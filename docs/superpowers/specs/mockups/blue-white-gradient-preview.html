视觉对比要点:

  • A (蓝→纯白):在白页面上失去"渐变形状",必须加边框或文字色重定义 — 改造成本最大,但深色模式下最惊艳
  • B (蓝→浅蓝 #B3D9FF):白底/深底通用,无需改文字色,直接替换 — 改造成本最低,推荐
  • C (主色→主色亮 5%):看上去几乎纯色,失去"渐变"语义,等于把"蓝紫渐变"降级成"纯主色"
  • D (深蓝→浅蓝白):与 AppColors.primary 脱钩,需要同步把 primary 也调成 #0059B3 才协调

30+ Web 文件硬编码改造的难易度:

  • B/C/D:单纯改 background 字符串就行
  • A:还要逐个改文字色 / 加边框 / 调整 box-shadow
